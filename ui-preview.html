日常使用工作流演示

看看 DotAgent 如何帮你轻松管理多个AI工具的配置

步骤 1: 创建/克隆项目
你用任意方式创建项目（git clone、手动新建等），不需要从本工具创建
$ git clone https://github.com/your/project.git
$ cd my-new-project
步骤 2: 初始化项目配置
在项目目录执行 agent-hub init，工具自动扫描现有配置、识别技术栈
$ agent-hub init
✓ 检测到现有配置: CLAUDE.md, AGENTS.md
✓ 识别技术栈: React + TypeScript + Vite
✓ 推荐模板: React/TS规范、Git规范
✓ 项目配置已创建
步骤 3: 调整配置（可选）
打开GUI，按需覆盖/禁用某些分段，添加项目特有规则，选择要用哪些工具
✓ 继承：AI角色定义 ✓ 继承：Git规范 ✏ 覆盖：开发工作流 🚫 禁用：Python规范 ⭐ 添加：项目架构说明
步骤 4: 一键同步
点击同步，自动在项目目录生成各工具的配置文件（含指令配置和API配置）
生成 CLAUDE.md → Claude Code 可识别
生成 AGENTS.md → Trae 可识别
生成 CODEX.md → Codex 可识别
生成 WORKBUDDY.md → WorkBuddy 可识别
API配置已同步到所有工具
步骤 5: 用你喜欢的工具开发
所有工具打开同一个项目文件夹，都能读到同一份规则！想用哪个用哪个！
